TECH STACK

FoxVault is built entirely on Google Cloud Platform. Every layer — compute, storage, encryption, auth, and observability — uses a managed GCP service with no self-hosted infrastructure.

⚡
Cloud Run Functions
Serverless compute for upload, download, and list operations. Runs Python 3.11 with per-request billing and VPC isolation.
Python 3.11
☁
Cloud Storage
Durable encrypted PDF storage with uniform IAM, 7-year retention policy, object versioning, and zero public access.
CMEK · Uniform IAM
🔑
Cloud KMS
Customer-managed AES-256 encryption keys. Dedicated key per vault with 365-day auto-rotation and service account binding.
AES-256-GCM
🌐
Cloud API Gateway
Managed HTTPS entry point with JWT validation via Identity Platform. All three function endpoints are secured behind the gateway.
JWT Auth
🔒
Identity Platform
Firebase-compatible email/password authentication. Issues JWTs consumed by API Gateway for all protected routes.
Email / Password
📋
Cloud Audit Logs
Data Read and Write events enabled on Cloud Storage and KMS. Every file access and decryption is logged automatically.
Data R/W Enabled
📊
BigQuery
Long-term compliance audit log archive via Log Router sink. Queryable 7-year audit trail for regulatory requirements.
Log Sink
📡
Pub/Sub
Security alerting message bus. Unauthorized access attempts trigger Pub/Sub messages routed to Cloud Monitoring email alerts.
Security Alerts
🛡
Serverless VPC Access
Private network connector ensuring Cloud Run functions communicate with GCP APIs over Google's internal network only.
Private Egress
